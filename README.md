1. How do I approach designing software?

Working on the 3D snowman scene taught me a lot about planning before jumping into code. Before I started building anything, I had to think about what shapes I would need, how they would fit together, and how the lighting would affect the final look. That kind of visual planning was new to me and honestly really helpful. I learned that breaking a complex scene down into smaller individual pieces, like treating the snowman as separate spheres stacked on top of each other, makes the whole thing way more manageable.

The design process I followed was to start simple and build up from there. I got the basic shapes on screen first, then added positioning, then lighting, and then textures last. Going in that order meant I always had something working to look at, which made it easier to spot problems early. I think this same approach of building in layers and testing as you go will carry over into any kind of software project I work on in the future, not just graphics.

2. How do I approach developing programs?

One of the biggest new strategies I used in this project was working with transformation matrices to position and scale objects in 3D space. At first that was really confusing, but once I understood that every object just needs a translation, rotation, and scale applied to it, it clicked and became a pattern I could repeat for every part of the scene. I also got a lot more comfortable reading and writing OpenGL code, which was completely new to me coming into this course.

Iteration was huge in this project. I never got anything right on the first try. I would place an object, run the program, see that it was in the wrong spot or the wrong size, go back and adjust the numbers, and run it again. Doing that over and over is how I figured out how everything worked. By the end of the milestones I was much faster at making those adjustments because I had a better mental picture of how the 3D coordinate system worked. My code also got cleaner over time as I started organizing things into functions instead of writing everything in one big block.

3. How can computer science help me in reaching my goals?

As someone who wants to become a software engineer, this course gave me skills I did not expect to be useful but absolutely are. Understanding how graphics pipelines work, how memory is managed when sending data to the GPU, and how transformation math works under the hood gives me a deeper understanding of how software interacts with hardware. A lot of modern software, from mobile apps to web interfaces to games, involves some kind of visual rendering, and now I have a foundation for understanding how that works.

On the professional side, having experience with OpenGL and 3D scene development sets me apart from other candidates who only know basic programming. Even if I do not end up working in graphics specifically, the problem solving skills I built here, like debugging visual output, working with coordinate systems, and managing complex state in a render loop, are all directly transferable to building real world software systems. This project showed me that I can take on something that feels overwhelming at first and work through it step by step until it is done.
